# weathereurope

An interactive map of European cities showing per-day weather (temperature + condition
icon) for a selectable date range. Capitals show first; smaller cities and towns reveal
as you zoom in. Built with Nuxt 4 + MapLibre GL, weather from
[Open-Meteo](https://open-meteo.com/) (MET Norway model), city data from
[GeoNames](https://www.geonames.org/).

## Development

```bash
pnpm install
pnpm dev          # http://localhost:3000
pnpm typecheck
```

## Architecture

- **`app/`** — Nuxt app. `WeatherMap.vue` renders MapLibre + DOM city markers;
  `DayRange.vue` is the from–to date-range picker.
- **`server/api/city-forecast.get.ts`** — given a viewport (`minLng,minLat,maxLng,maxLat,zoom`),
  selects the cities to show and returns them as GeoJSON with an 8-day forecast each.
- **`server/utils/cities.ts`** — zoom→population tiers (capitals always shown), capped at 120 cities.
- **`server/utils/openMeteo.ts`** — bulk forecast fetch with a stale-while-error cache.
- **`server/utils/rateLimit.ts`** — per-IP request limiter.
- **`server/data/cities.json`** — bundled dataset, generated by `scripts/build-cities.mjs`.

### Regenerating the city data

```bash
pnpm build:cities   # re-downloads GeoNames cities15000 and rewrites server/data/cities.json
```

Requires `curl` and `unzip` on PATH. GeoNames data is CC BY 4.0 and is attributed in the UI.

## Deploying to Vercel

Nuxt auto-detects Vercel — no `vercel.json` or preset needed. Import the repo and deploy.

### Environment variables

| Variable | Purpose | Required |
| --- | --- | --- |
| `UPSTASH_REDIS_REST_URL` | Shared cache + rate-limit store (REST) | Recommended in prod |
| `UPSTASH_REDIS_REST_TOKEN` | Upstash auth token | Recommended in prod |
| `NUXT_OPEN_METEO_CONTACT` | Contact string sent in the Open-Meteo `User-Agent` (use a role address) | Recommended |

**Why Upstash:** on serverless the default cache is per-instance in-memory — wiped on cold
start and not shared, so hit rates collapse and every viewport re-hits Open-Meteo. Set the
two `UPSTASH_*` vars (any Upstash Redis database, free tier is fine) and both the forecast
cache and the rate limiter become shared and durable. Without them the app still runs, just
with a weaker (per-instance) cache — fine for local dev. To use Vercel KV or another store
instead, swap the driver in `nuxt.config.ts` (`nitro.storage`).

### Notes / limitations

- **Open-Meteo terms:** the free tier is non-commercial. A public production deployment
  should use their commercial plan or a self-hosted instance.
- **Rate limiting** is a best-effort abuse backstop (fixed window, non-atomic), currently
  60 req/min per IP — tune in `city-forecast.get.ts`.
- **CDN caching:** the API sends `s-maxage`/`stale-while-revalidate`, so Vercel's edge caches
  identical viewport URLs. Client viewports use high-precision coordinates, so cross-user
  hit rates are modest — quantizing the client's bbox/zoom would improve this.
